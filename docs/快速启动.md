# 快速启动指南

## 前置要求

- Docker & Docker Compose
- 至少一个 LLM API Key（OpenAI / DeepSeek / 通义千问等）
- 豆包 Embedding API Key（用于文档向量化）

---

## 1. 配置环境变量

复制示例文件并填写必要配置：

```bash
cp .env.example .env
```

编辑 `.env`，**必填项**：

```env
# Embedding（必填，用于文档向量化）
DOUBAO_API_KEY=your-doubao-api-key
DOUBAO_EMBEDDING_MODEL=your-embedding-endpoint-id
DOUBAO_HOST=https://ark.cn-beijing.volces.com

# LLM（至少填写一个）
OPENAI_KEY=sk-xxx
DEEPSEEK_KEY=sk-xxx
QWEN_API_KEY=sk-xxx

# JWT 密钥（生产环境必须修改）
JWT_SECRET=your-secret-key
```

---

## 2. 启动服务

```bash
docker-compose up -d
```

启动后包含以下服务：

| 服务 | 地址 |
|------|------|
| API 服务 | http://localhost:4001 |
| MySQL | localhost:3307 |
| Redis | localhost:6380 |
| Meilisearch | http://localhost:7701 |
| **测试平台 UI** | **http://localhost:4001/test** |

> 前端为静态 HTML，由 API 服务直接托管，无需单独启动。

验证服务正常：

```bash
curl http://localhost:4001/health
# 返回: {"status": "ok"}

curl http://localhost:4001/api/health
# 返回: {"status": "ok", "db": true, "redis": true, "meilisearch": true}
```

## 3. 核心使用流程

### 3.1 创建知识库

```bash
curl -X POST http://localhost:4001/api/rag/kb \
  -H "Content-Type: application/json" \
  -d '{
    "name": "建筑规范库",
    "description": "中国建筑设计规范文档",
    "embedding_model": "doubao"
  }'
```

返回示例：`{"code": 0, "data": {"id": 1, "name": "建筑规范库", ...}}`

记录返回的 `id`，后续步骤使用。

---

### 3.2 上传文档

支持格式：`.md`、`.txt`、`.pdf`、`.docx`、`.xlsx`

```bash
curl -X POST http://localhost:4001/api/rag/document/upload \
  -F "kb_id=1" \
  -F "file=@/path/to/your-document.md" \
  -F "chunk_size=500" \
  -F "chunk_overlap=50"
```

上传后系统自动在后台进行：切片 → 向量化 → 建索引

查看处理状态：

```bash
curl http://localhost:4001/api/rag/document/{doc_id}
# status: pending → processing → completed
```

---

### 3.3 搜索知识库

文档处理完成后即可搜索：

```bash
curl -X POST http://localhost:4001/api/rag/search \
  -H "Content-Type: application/json" \
  -d '{
    "kb_id": 1,
    "query": "防火间距要求",
    "top_k": 5,
    "threshold": 0.3
  }'
```

---

### 3.4 RAG 问答（流式）

```bash
curl -X POST http://localhost:4001/api/ppt/stream/rag-query \
  -H "Content-Type: application/json" \
  -d '{
    "message": "住宅建筑的防火间距有哪些规定？",
    "kb_ids": [1],
    "model_id": "your-model-id"
  }'
```

响应为 SSE 流，依次返回：

```
data: {"type": "metadata", "conversation_id": "xxx", ...}

data: {"type": "search_results", "chunks": [...]}

data: 根据《建筑设计防火规范》...（逐 token 输出）

data: [DONE]
```

## 4. LLM 配置（可选）

系统内置 OpenAI / Anthropic / Gemini / DeepSeek 等提供商，需通过 API 配置 API Key：

```bash
# 更新提供商 API Key
curl -X PUT http://localhost:4001/api/llm/provider/{provider_id}/api-key \
  -H "Content-Type: application/json" \
  -d '{"api_key": "sk-xxx"}'

# 查看可用模型
curl http://localhost:4001/api/llm/model/by-provider
```

## 常见问题

**Q: 上传文档后 status 一直是 pending？**
检查 Embedding 配置是否正确：`DOUBAO_API_KEY` 和 `DOUBAO_EMBEDDING_MODEL` 是否有效。

**Q: 搜索没有结果？**
确认文档 status 已变为 `completed`，并且 `threshold` 不要设置过高（建议 0.3 以下）。

**Q: SSE 流无响应？**
检查 `model_id` 是否存在且对应的提供商 API Key 已配置。通过 `/api/llm/model` 查看可用模型列表。
