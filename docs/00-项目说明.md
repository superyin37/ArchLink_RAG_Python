# 00 - 项目说明

## 项目简介

本项目是一套由 Node.js/Koa 生产系统重构而来的 **Python 版 RAG（检索增强生成）系统**，以 FastAPI 为核心的单容器后端应用，提供：

- **知识库管理**：知识库/文档 CRUD、文档树可视化、导入导出迁移
- **文档处理流水线**：加载 → 解析 → 切片（树状结构）→ 嵌入 → 索引（向量 + 全文）
- **混合检索**：向量搜索（LanceDB）+ 全文搜索（Meilisearch）+ RRF/加权融合、去重、阈值自适应、可读性重排、Token 预算压缩
- **多提供商 LLM 接入**：OpenAI / Anthropic / Azure OpenAI / Google Gemini / 火山引擎（Volcengine），统一流式（SSE）输出、调用日志、费用统计、重试与回退
- **PPT 生成模块**：基于阶段状态机的多轮对话，结合 RAG 检索编排完成大纲/案例/内容生成

## 架构总览

- **分层**：Router（FastAPI 路由）→ Service（业务）→ Repository/Model（SQLAlchemy ORM）
- **模块化**：`app/modules/{rag, llm, ppt}`，模块间仅通过服务接口通信
- **模块依赖**：`ppt → llm / rag`，`rag → llm（可选，用于文档名预过滤）`，`llm` 独立
- **流式传输**：统一采用 Server-Sent Events（SSE），不使用 WebSocket
- **响应格式**：`{ code, msg, data }` 由统一 `R` 包装器产出
- **部署形态**：Docker + docker-compose，单应用容器（uvicorn），外挂 MySQL / Redis / Meilisearch

## 技术栈

### 运行时与语言
| 项 | 选型 |
|---|---|
| 语言 | Python 3.11+ |
| 异步模型 | asyncio / async-await（全链路 I/O 异步） |
| 容器化 | Docker + docker-compose |
| 进程 | uvicorn（`uvicorn[standard]`） |

### 存储与基础设施
| 类别 | 选型 | 说明 |
|---|---|---|
| 关系数据库 | **MySQL 8.0** | utf8mb4，通过 `aiomysql` 异步驱动访问 |
| 向量数据库 | **LanceDB** + **pyarrow** | 本地目录存储（`database/lancedb/`），snake_case 列名；pyarrow 用于 Arrow schema/表构造 |
| 全文检索 | **Meilisearch v1.6** | 索引命名规则 `kb_{kb_id}` |
| 缓存/限流 | **Redis 7** | `redis[hiredis]` 异步客户端，提供内存回退 |
| 文件存储 | 本地 `uploads/` | 通过 `StaticFiles` 挂载到 `/uploads` |

### Web 框架与核心库
| 库 | 用途 |
|---|---|
| **FastAPI** (≥0.110) | Web 框架、依赖注入、OpenAPI |
| **sse-starlette** | SSE 流式响应 |
| **python-multipart** | 文件上传 |
| **SQLAlchemy 2.0**（`[asyncio]`） | 异步 ORM，`async_sessionmaker` |
| **Alembic** | 数据库迁移 |
| **Pydantic v2** + **pydantic-settings** | 请求/响应模式、基于环境变量的配置管理 |
| **python-dotenv** | `.env` 支持 |
| **aiofiles** | 异步文件 I/O |
| **httpx** | 通用异步 HTTP 客户端（含 Doubao 嵌入、通用 LLM HTTP 调用） |

### LLM 提供商 SDK
| 库 | 适配器 |
|---|---|
| `openai` | OpenAI / Azure OpenAI（`completions/openai.py`, `azure.py`） |
| `anthropic` | Anthropic（含 prompt 缓存） |
| `google-generativeai` | Google Gemini |
| `httpx` | DeepSeek / Volcengine / 通用 HTTP 提供商 |

补充能力：自动注册工厂（`LLMOne` / `ResponseOne`）、`withRetryBackoff` 重试、`withFallbackRouter` 回退、调用生命周期日志、Token 消耗与费用计算。

### 嵌入与文档解析
| 库 | 用途 |
|---|---|
| Volcengine **Doubao** Embedding API（`httpx` 调用） | 向量嵌入生成 |
| **python-docx** / **mammoth** | DOCX 解析 |
| **PyPDF2**（备选 pdfplumber） | PDF 解析 |
| **openpyxl** | XLSX 问答对加载 |
| **markdown** | Markdown 解析 |

文档经自定义 `chunker` 按树结构切片（材料化路径 `0001/0002/0003`），支持子/兄弟/祖先召回增强。

### 检索管线组件（`app/modules/rag/search/`）
融合（RRF / 加权 / 线性）、去重（ID + 内容 + 父子节点）、阈值自适应、可读性重排、关键词提取、文档名预过滤、Token 预算上下文压缩、树上下文组装。

### 认证与安全
| 库 | 用途 |
|---|---|
| **PyJWT** | JWT 签发/解析 |
| **bcrypt** | 密码哈希 |
| 自定义中间件 | `auth.py`（白名单）、`error_handler.py`、`request_context.py`（contextvars）、`response_wrapper.py` |

### 前端（测试 UI，`static/test-ui/`）
| 依赖 | 用途 |
|---|---|
| **Vue 3**（Composition API，本地 prod build） | 单文件 UI |
| **Tailwind CSS**（本地 vendor） | 样式 |
| **marked.js** | Markdown 渲染 |
| **KaTeX** | 数学公式 |

无构建工具，整个测试前端打包在单个 HTML 中，挂载于 `/test/`。

### 数值计算
| 库 | 用途 |
|---|---|
| **numpy** | 向量运算、检索评分等（当前作为 LanceDB/pyarrow 的传递依赖随 LanceDB 安装，`requirements.txt` 未显式声明，如后续直接使用建议显式加入） |

## 关键约定

- 全链路 **async/await**，禁止阻塞调用
- 类型注解必填，Pydantic v2 约束请求/响应
- 路径统一用 `pathlib.Path`
- 时间统一 **UTC**，`DateTime` 存储
- **软删除** 通过 `delete_time IS NULL` 判定
- 单文件代码行数约束在 300 行以内，超出拆分子模块
- 配置全部走环境变量（参见 `.env.example` / `app/config.py`）
