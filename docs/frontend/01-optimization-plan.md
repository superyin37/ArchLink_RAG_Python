# 前端优化方案

> 文件路径：`static/test-ui/index.html`
> 访问地址：`http://localhost:4001/test/`

---

## 一、交互逻辑优化

### 1.1 默认页面
- 启动时默认显示 **RAG 对话** tab，而非搜索对比
- 修改方式：`activeTab` 初始值从 `'search'` 改为 `'chat'`

### 1.2 知识库 & 模型记忆
- 选择知识库/模型后写入 `localStorage`，下次打开自动恢复
- 页面加载完成后，如果 localStorage 有记录且该 KB/模型仍在列表里，自动选中
- 知识库列表加载完成后，若无记忆值则自动选第一条

### 1.3 Token 持久化（已有，确认保留）
- API Base 和 Token 已经持久化，继续保留

## 二、UI 设计优化

### 2.1 整体布局

| 区域 | 现状 | 优化 |
|------|------|------|
| Header | 配置项全堆一排，拥挤 | API/Token 收进折叠面板或设置抽屉，Header 只留 Logo + 健康状态 |
| Tab 栏 | 纯文字 tab | 加图标（💬 对话 / 🔍 搜索 / 📚 知识库） |
| 聊天气泡 | 最大宽度 `max-w-xl` 固定 | 改为百分比，适应宽屏 |

### 2.2 RAG 对话页
- **配置栏折叠**：知识库/模型/TopK 默认收起，右上角「配置」按钮展开，减少视觉噪音
- **气泡宽度**：AI 回复气泡扩展到容器 85%，用户气泡保持右对齐
- **流式光标**：回复生成中在末尾显示闪烁光标 `▍`
- **代码块复制按钮**：代码块右上角加「复制」按钮
- **消息时间戳**：气泡下方小字显示发送时间
- **RAG 上下文面板**：chunk 列表支持展开/折叠全文（现在截断了看不全）

### 2.3 搜索对比页
- 搜索结果 chunk 卡片支持点击展开全文
- 分数条颜色区分（已有，保留）

### 2.4 KB 概览页
- 知识库卡片加「去对话」快捷按钮，点击自动切换到对话 tab 并选中该 KB

## 三、细节修复

| 问题 | 方案 |
|------|------|
| 流式输出期间 Markdown 渲染抖动（标题/列表不断重排） | 流式中只渲染纯文本 `innerText`，停止输出 500ms 后再渲染 Markdown |
| 空消息占位符「`...`」被 Markdown 渲染 | 单独处理占位状态，显示骨架动画而非渲染省略号 |
| RAG 上下文 chunk 分数太小（0.03）视觉上都接近 0 | 分数条改为相对归一化（最高分 = 100%） |
| 页面标题固定「RAG 测试平台」 | 加上当前知识库名称，如「RAG · 建筑规范-Qwen」 |
